# Work Report

git log 기반으로 기간별 작업 내용을 분석하여 요약 리포트를 생성한다.

## 핵심 제약사항

- 코드를 수정하지 않는다. 리포트 생성만 수행한다.
- git log에 기록된 사실만 기반으로 작성한다. 추측하지 않는다.
- 커밋 메시지가 불명확한 경우 diff를 참고하여 보완한다.
- 출력 형식은 `references/report-template.md`를 따른다.

## 리포트 유형

| 유형 | 기간 | 용도 |
|------|------|------|
| 일일 | 오늘 (당일) | 일일 작업 보고, 스탠드업 |
| 주간 | 이번 주 (월~금) | 주간 보고, 회고 |
| 커스텀 | 사용자 지정 기간 | 특정 기간 작업 정리 |

## 워크플로우

1. 사용자에게 기간 확인 (오늘 / 이번 주 / 커스텀 기간)
2. `git log --author`로 해당 기간 내 커밋 조회
   - 여러 프로젝트가 있는 경우 각각 조회
3. 커밋을 프로젝트별로 그룹화
4. 각 커밋을 작업 유형별로 분류 (feat, fix, refactor 등)
5. 커밋 메시지만으로 부족한 경우 `git diff`로 변경 내용 보완
6. `references/report-template.md` 형식에 맞춰 리포트 출력

## 참조 문서

| 문서 | 용도 | 언제 읽는가 |
|------|------|-----------|
| `references/report-template.md` | 리포트 출력 템플릿 | 리포트 작성 시 출력 형식을 결정할 때 |
