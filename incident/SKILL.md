# Incident

장애 발생 시 로그 검색 → 원인 분석 → 영향 범위 파악 → 대응/롤백 가이드를 하나의 플로우로 제공한다.

## 핵심 제약사항

- 운영 환경에 직접 명령을 실행하지 않는다. 가이드만 제공한다.
- 추측으로 원인을 단정하지 않는다. 로그/데이터 기반으로 분석한다.
- 롤백/핫픽스는 가이드만 제공하고, 실행은 사용자가 판단한다.
- 장애 대응 절차는 `references/response-playbook.md`를 따른다.

## 대응 범위

| 단계 | 설명 | 도구 |
|------|------|------|
| 증상 파악 | 장애 증상, 시작 시점, 영향 범위 확인 | 사용자 인터뷰 |
| 로그 검색 | 에러 로그, 요청 로그 분석 | log-search 패턴 (DB + ES) |
| 원인 분석 | 로그 기반 근본 원인 추적 | 코드 분석 + 로그 상관관계 |
| 영향 범위 | 영향받은 사용자 수, API, 기능 파악 | DB 쿼리 + 로그 집계 |
| 대응 방안 | 즉시 조치 + 근본 해결 방안 제시 | 코드 분석 |
| 롤백 가이드 | 필요시 롤백 절차 제공 | git log + 배포 이력 |

## 워크플로우

1. 사용자에게 장애 증상, 시작 시점, 영향 범위 확인
2. `references/response-playbook.md`를 참조하여 대응 절차 확인
3. log-search 패턴으로 관련 로그 검색 (DB HTTP_LOG + ES)
   - 에러 로그 필터링 (5xx, 타임아웃 등)
   - 시간대별 에러율 추이 확인
4. 로그 기반 원인 분석
   - 에러 발생 코드 위치 추적
   - 최근 배포/변경사항과 상관관계 분석
5. 영향 범위 파악
   - 영향받은 사용자 수/API 집계
   - 비즈니스 임팩트 정리
6. 대응 방안 출력
   - 즉시 조치 사항 (임시 조치)
   - 근본 해결 방안 (코드 수정)
   - 롤백 필요시 롤백 가이드
