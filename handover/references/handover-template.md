## 태스크 문서

## 작업 배경

- 이 작업을 하게 된 배경, 요구사항, 관련 회의/논의 내용 정리
- 비즈니스 컨텍스트 (어떤 기능이 왜 필요한지)

## 주요 작업 내용

### {도메인/모듈 1}

- 관련 폴더/파일 참고 안내
- 핵심 변경 사항 요약

### {정책/기능 1} 정책

- 기준일자 명시 (예: 대한항공 기준 20260107)
- 금액, 수량, 조건 등 구체적 정책 내용
- 서비스별 적용 규칙
    - 서비스 A - 적용 방식 설명
    - 서비스 B - 적용 방식 설명
- 어드민 설정 가능 항목

### {정책/기능 2} 정책

- 트랜잭션/상태 변경 규칙
- 타입별 동작 (예: USE, CHANGE, REGISTER)
- 우선순위 규칙 (있는 경우)
    1. 우선순위 1
    2. 우선순위 2
    3. 우선순위 3

### {정책/기능 3} 정책

- 환불/취소 등 예외 케이스 정책
- 시간대별/조건별 규칙 표
    - 조건 A: N% 적용
    - 조건 B: N% 적용
    - 조건 C: 0% 적용
- 혼합 사용 규칙 (있는 경우)
- 이용내역 설명 필드 포맷
    - 도메인 A
        - **사용** `{id}` 설명
        - **롤백** `{id}` 설명
        - **전환** `{id}` 설명
- 알림 발송 규칙 (있는 경우)

### ~~취소선 섹션~~

- 기획 변경이나 더 이상 유효하지 않은 내용은 취소선 처리하여 이력 보존
- 변경 사유 및 관련 스레드 링크 첨부

## 기타

- 테스트 도구 (포스트맨 컬렉션 등)
- 초기 요구명세 링크
- 관련 회의록 링크

## 백로그

- 아직 완료되지 않은 후속 작업
- 알려진 제한사항 및 개선 필요 사항
- 기존 시스템과의 정합성 이슈
