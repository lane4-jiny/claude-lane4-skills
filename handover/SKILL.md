# Handover

주제를 기반으로 git log, diff, 코드 분석을 통해 작업 배경, 변경점, 설계 결정 이유를 정리한 인수인계 문서를 생성한다.

## 핵심 제약사항

- 코드를 수정하지 않는다. 문서 생성만 수행한다.
- 추측으로 작성하지 않는다. git 이력과 코드에서 확인된 사실만 기록한다.
- 변경 이유가 불명확한 경우 "확인 필요"로 표기한다.
- 출력 형식은 `references/handover-template.md`를 따른다.

## 분석 대상

| 항목 | 소스 | 설명 |
|------|------|------|
| 작업 배경 | 커밋 메시지, 브랜치명 | 왜 이 작업을 했는지 |
| 변경 파일 | git diff | 어떤 파일이 변경되었는지 |
| 변경 내용 | diff + 코드 읽기 | 무엇이 바뀌었는지 |
| 설계 결정 | 코드 구조 분석 | 왜 이렇게 구현했는지 |
| 주의사항 | 코드 분석 | 인수인계 시 유의할 점 |

## 워크플로우

1. 사용자에게 주제 또는 브랜치/커밋 범위 확인
2. `git log`로 관련 커밋 이력 조회
3. `git diff`로 변경사항 전체 파악
4. 변경 파일별로 무엇이/왜 바뀌었는지 코드 분석
5. 설계 결정 포인트 식별 및 이유 정리
6. 주의사항 및 후속 작업 정리
7. `references/handover-template.md` 형식에 맞춰 인수인계서 출력

## 참조 문서

| 문서 | 용도 | 언제 읽는가 |
|------|------|-----------|
| `references/handover-template.md` | 인수인계서 출력 템플릿 | 인수인계서 작성 시 출력 형식을 결정할 때 |
