# Code Review

PR이나 변경사항에 대해 코드 리뷰를 수행한다. 문제를 사전에 예방하고 코드 품질을 유지한다.

## 핵심 제약사항

- 코드를 직접 수정하지 않는다. 리뷰 피드백만 제공한다.
- 사용자가 수정을 요청한 경우에만 코드를 변경한다.
- 추측성 피드백은 제공하지 않는다. 코드에서 확인된 문제만 지적한다.
- 변경된 코드만 리뷰한다. 기존 코드의 문제는 별도 언급한다.

## 리뷰 범위

### 1. NestJS 패턴 위반
- 트랜잭션 누락 (다중 INSERT/UPDATE 시)
- Guard/Decorator 미설정
- DTO 검증 누락 (class-validator)
- Repository 직접 호출 (Service 우회)
- 순환 의존성

### 2. 보안 취약점
- SQL Injection 가능성
- 인증/인가 누락
- 민감 정보 노출 (로그, 응답)
- 하드코딩된 시크릿

### 3. 성능
- N+1 쿼리
- 불필요한 전체 조회 (findAll without pagination)
- 누락된 인덱스 힌트
- 대용량 데이터 메모리 로드

### 4. 컨벤션
- lane4 프로젝트 네이밍 규칙 위반
- 파일/클래스 구조 불일치
- 공통 패턴 미사용

## 심각도 분류

| 심각도 | 설명 | 예시 |
|--------|------|------|
| 🔴 Critical | 배포 차단. 반드시 수정 필요 | 트랜잭션 누락, 보안 취약점, 데이터 유실 가능성 |
| 🟡 Warning | 수정 권장. 잠재적 문제 | N+1 쿼리, 에러 핸들링 부족, 타입 미스매치 |
| 🔵 Info | 참고사항. 개선 제안 | 네이밍 개선, 코드 구조 제안, 주석 추가 |

## 리뷰 결과 형식

```
## 코드 리뷰 결과

### 요약
- 변경 파일: N개
- Critical: N건 / Warning: N건 / Info: N건

### 🔴 Critical
1. [파일명:라인] 내용
   - 문제: ...
   - 제안: ...

### 🟡 Warning
1. [파일명:라인] 내용

### 🔵 Info
1. [파일명:라인] 내용

### 전체 의견
...
```

## 워크플로우

1. `git diff` 또는 `git diff --staged`로 변경사항 확인
2. 변경된 파일 목록 파악 및 변경 유형 분류
3. `references/conventions.md`를 참조하여 lane4 프로젝트 컨벤션 확인
4. `references/review-checklist.md`를 참조하여 체크리스트 기반 리뷰 수행
5. 각 파일의 변경 코드를 읽고 문제점 분석
6. 심각도별로 분류하여 리뷰 결과 형식에 맞춰 출력
7. Critical 항목이 있으면 배포 전 수정 필요 명시

## 참조 문서

| 문서 | 용도 | 언제 읽는가 |
|------|------|-----------|
| `references/review-checklist.md` | NestJS 패턴, 보안, 성능, 컨벤션 체크리스트 | 리뷰 항목을 빠짐없이 확인할 때 |
| `references/conventions.md` | lane4 프로젝트 코딩 컨벤션 | 네이밍, 구조, 패턴 위반 여부 판단할 때 |
