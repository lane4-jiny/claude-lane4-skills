# Deploy

개발계 → 운영계 배포 전 사이드이펙트를 확인한다. 코드 변경, DB 스키마 변경, 외부 연동 변경을 종합 분석하여 배포 가능 여부를 판단한다.

## 핵심 제약사항

- 코드를 수정하지 않는다. 분석 결과만 제공한다.
- 실제 배포를 수행하지 않는다.
- 불확실한 영향은 "확인 필요"로 표시한다.
- DB 운영 데이터를 변경하지 않는다 (SELECT만 사용).

## 분석 항목

### 1. 코드 변경사항 분석
- `git diff dev..main` (또는 지정된 브랜치) 기반 변경 파일 목록
- 변경 유형 분류: 신규 API, API 수정, 로직 변경, 설정 변경
- Breaking Change 여부 판단

### 2. DB 스키마 변경 분석
- Entity/Migration 파일 변경 감지
- 새 테이블/컬럼 추가, 컬럼 타입 변경, 인덱스 변경
- lane4-mysql MCP로 현재 운영 스키마와 비교 (DESCRIBE, SHOW CREATE TABLE)
- 데이터 마이그레이션 필요 여부

### 3. 외부 연동 변경 분석
- .env 파일 변경 (새 환경변수 추가, 값 변경)
- 외부 API 연동 변경 (엔드포인트, 인증, 파라미터)
- Firebase, Redis, Elasticsearch 설정 변경

### 4. 프로젝트 간 의존성 분석
- 변경된 테이블을 사용하는 다른 프로젝트 확인
- 공유 라이브러리/모듈 변경시 영향 범위
- API 변경시 호출하는 다른 서비스 확인

## 결과 분류

| 분류 | 의미 | 조건 |
|------|------|------|
| ✅ 배포 가능 | 안전하게 배포 가능 | 블로커 없음, 주의사항 확인 완료 |
| ⚠️ 주의 | 배포 가능하나 확인 필요 | 수동 확인 필요 항목 존재 |
| 🚫 블로커 | 배포 전 해결 필요 | Breaking Change, 마이그레이션 미완료, 환경변수 미설정 |

## 결과 출력 형식

```
## 배포 전 체크 결과

### 판정: ✅ 배포 가능 / ⚠️ 주의 / 🚫 블로커

### 코드 변경 요약
- 변경 파일: N개
- 신규 API: N개
- 수정 API: N개
- 주요 변경: ...

### DB 스키마 변경
- [ ] 새 테이블: ...
- [ ] 새 컬럼: ...
- [ ] 마이그레이션: ...

### 외부 연동 변경
- [ ] 새 환경변수: ...
- [ ] 연동 변경: ...

### 프로젝트 간 영향
- [ ] 영향받는 프로젝트: ...

### 배포 순서 권장
1. ...
2. ...

### 배포 후 확인사항
- [ ] ...
```

## 워크플로우

1. 배포 대상 브랜치/커밋 범위 확인 (사용자에게 질문)
2. `git diff`와 `git log`로 코드 변경사항 분석
3. `references/deploy-checklist.md`를 참조하여 체크리스트 항목 점검
4. Entity/Migration 변경시 lane4-mysql MCP로 현재 DB 스키마 비교
5. `references/cross-project-deps.md`를 참조하여 프로젝트 간 영향 분석
6. .env / 설정 파일 변경사항 확인
7. 결과를 종합하여 배포 판정 및 체크리스트 출력

## MCP 도구 활용

| 도구 | 용도 |
|------|------|
| lane4-mysql | DESCRIBE, SHOW CREATE TABLE로 운영 스키마 확인 |
| git (기본) | git diff, git log로 코드 변경 분석 |

## 참조 문서

| 문서 | 용도 | 언제 읽는가 |
|------|------|-----------|
| `references/deploy-checklist.md` | 배포 전 체크리스트 항목 | 체크리스트 기반 점검할 때 |
| `references/cross-project-deps.md` | 프로젝트 간 공유 테이블/연동 의존성 | 프로젝트 간 영향 분석할 때 |
