# 카테고리 분류 예시

커밋과 캘린더 일정을 카테고리로 분류할 때 참고한다.

## 카테고리 추출 기준

### 1. 커밋 메시지에서 키워드 추출
- `FARE_HISTORY` 관련 → **FARE_HISTORY**
- `호출자`, `caller` 관련 → **호출자**
- `안심번호`, `safe number` 관련 → **안심번호**
- `어드민`, `admin` 관련 → **어드민**
- `인스파이어`, `inspire` 관련 → **인스파이어**
- `charter` 관련 → **charter**
- `EK`, `메일 파싱` 관련 → 독립 항목 또는 기타
- `배차`, `allocation` 관련 → **배차**
- `알림`, `notification` 관련 → **알림**

### 2. 변경 파일 경로에서 추출
- `fare-history/` → **FARE_HISTORY**
- `admin/` → **어드민**
- `caller/` → **호출자**
- `notification/` → **알림**

### 3. 카테고리가 2개 이상 항목을 가질 때만 독립 카테고리로 분리
- 1개 항목만 있는 주제 → "기타" 카테고리로 통합
- 단, 중요도가 높은 작업은 1개여도 독립 카테고리 가능

## 항목 작성 스타일

### Good
- `법인어드민 요금 예외처리 리스트업`
- `요금 조회 에러로 인한 0원 처리 시 슬랙 알림 추가`
- `인스파이어 당일 조건 KST 변환 누락 수정 배포`
- `개발계 테스트`
- `charter 관련 레거시 파악`

### Bad
- `feat: add fare history save logic` (영어 커밋 메시지 그대로)
- `FARE_HISTORY 모듈의 saveFareHistory 메서드에 법인별 요금 동기화 로직 추가` (너무 기술적)
- `작업함` (너무 모호)
